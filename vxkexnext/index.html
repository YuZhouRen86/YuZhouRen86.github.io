介绍

VxKex NEXT 是一套适用于 Windows 7 的 API 扩展，可让一些 Windows 8、8.1 和 10 独占应用程序在 Windows 7 上运行。

安装前，建议执行以下操作。

  • 卸载
    • 0patch Agent
      它可能导致基于 Chromium 的浏览器和 JetBrains IDE 在启用 VxKex NEXT 并运行后崩溃。
  • 更新
    • MacType → 2025.6.9+
      旧版 MacType 可能导致所有程序在启用 VxKex NEXT 后无法启动。

安装后，使用很简单。以下是启用 VxKex NEXT 的方法：
1. 右键单击程序，打开属性对话框，选择“VxKex”选项卡。然后，选中“为此程序启用 VxKex NEXT”复选框，并尝试运行程序。
2. 从开始菜单中找到“VxKex NEXT Global Settings”并打开，点击“添加”按钮，选择程序，点击“打开”按钮，并尝试运行程序。

有些程序需要额外配置。VxKex NEXT 安装文件夹（默认为 C:\Program Files\VxKex 文件夹）内有一个名为“Application Compatibility List.docx”的文件，其中详细说明了这些步骤，但大多数情况下，所有配置都是不言自明的。

常见问题

哪些应用程序受支持？

大量现代程序在启用 VxKex NEXT 后可以在 Windows 7 上正常运行。请参阅与 VxKex NEXT 一起安装的Application Compatibility List.docx文件。

VxKex NEXT 会修改系统文件吗？它会使我的系统不稳定吗？

VxKex NEXT 不会修改任何系统文件。它对整个系统的影响极小。不使用后台服务，不安装全局钩子，加载的 shell 扩展和 DLL 影响也很小，可以随时禁用。您可以放心，您的 Windows 7 将一如既往地保持稳定。

我需要安装特定的更新吗？

没有任何更新的 Windows 7 用户仍可使用它，但许多程序需要安装 Service Pack 1、KB2533623（DllDirectories 更新）和 KB2670838（平台更新）才能运行。最好安装这些更新。

如果我安装了 ESU（扩展安全更新），我可以使用 VxKex NEXT 吗？

是的，ESU 没有问题。

可以在 Windows 8 或 8.1 中使用吗？

目前，VxKex NEXT 只适用于 Windows 7。如果您使用的是 Windows 8 或 8.1，VxKex NEXT 将毫无用处。

升级到 Windows 8/8.1/10/11 后可以删除 VxKex 或 VxKex NEXT 吗？

可以。如果 VxKex 已安装，请将其更新为 VxKex NEXT，然后从控制面板卸载它。

VxKex NEXT 如何运作？

VxKex NEXT 的工作原理是在启用 VxKex NEXT 的每个程序中加载一个 DLL。这是通过使用 IFEO（图像文件执行选项）注册表键来实现的。

具体来说，“VerifierDlls”值被设置为指向 VxKex NEXT DLL。该 DLL 会加载到进程中。

API 扩展是通过编辑程序的动态链接库导入表来实现的，这样程序就不会从 Windows 8/8.1/10/11 动态链接库中导入，而是导入 VxKex NEXT 动态链接库。这些 VxKex NEXT 动态链接库包含较新版本 Windows 中引入的 Windows API 函数的实现。
